DNS Configuration

Website is down - here are the troubleshooting steps to bringing it back to life!

First, let's eliminate the possibility that the site is down due to a misconfiguration on the client's end.

Open up a terminal and run the following command:

Windows: nslookup www.domainname.com

Mac: dig www.domainname.com

Core
  1. A record: The client's A record (@ host) should be pointing at the Core server IP address - 72.46.233.151
  2. WWW CNAME: The client's WWW CNAME record should be pointing at "@"; the A record, which should be pointing at Core's IP

Cloud
  1. A record: This one is tricky - all we care about here is that domainname.com is forwarding to www.domainname.com. You can test this by navigating to http://domainname.comâ€‹ (with the http://) bar and see if it automatically redirects you to http://www.domainname.com
    1. IF the naked domain (without the www.) isn't forwarding to www.domainname.com, do the following
      1. Determine what registrar the domain is hosted on. To do this, navigate to your terminal and run the following command: whois domainname.com
      2. This should return quite a bit of info about the domain, including which registrar it's using. Knowing which registrar the domain is on is going to help a lot with the following step.
      3. A domain forward needs to be placed on the naked domain (domainname.com without the "www.") that forwards to www.domainname.com. This is a setting that is available in most registrars. It's typically pretty easy to find an instruction guide to setting up a domain forward for the registrar at hand.
  2. WWW CNAME: Should point to the client's vanity domain - domainname.g5dns.com. (e.g. If they owned www.google.com, they would need to enter google.g5dns.com)

Core Configuration
  1. First, take a look at the location's configuration in the Client Center
    1. The `Status` must be set to Active
    2. The `Website Display` box must be checked
    3. While we're here, let's make note of the store id that displays in between the tabs and the Store Name field, in this case, 3772
  2. Next, let's take a look at the `Domains` tab at the Client level
    1. Find the domain in question inside of the `Domains` tab and check the following
      1. Is the domain correct? (shouldn't have the www.)
      2. Is the store ID correct? (same id we took note of above)

Cloud Configuration
  • Heroku or AWS?
    • To identify whether a site is hosted on Heroku or AWS, simple hover over the "View" link in the CMS for any location.
      • If the URL structure ends with herokuapp.com - it's on Heroku
      • If the URL structure ends with g5static.com - it's on AWS
